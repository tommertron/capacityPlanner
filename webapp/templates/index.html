Portfolio Planner - Help Guide

Overview

Portfolio Planner is a capacity planning tool that helps you manage project portfolios, allocate resources, and visualize team workload over time. The system uses a scheduling algorithm to assign people to projects based on their roles, skills, availability, and project priorities.

Getting Started

1. Create or Select a Portfolio

  • Create New Portfolio: Click the "+ New Portfolio" button next to the portfolio selector. Enter a name (letters, numbers, dashes, and underscores only). The new portfolio will be created with sample data that you can customize.
  • Select Portfolio: Use the dropdown menu at the top of the page to select an existing portfolio to work with.

2. Configure Your Portfolio Data

Once a portfolio is selected, you can configure the following:

Projects Tab

Projects Subtab

  • View Projects: See all projects in your portfolio with their effort estimates, priorities, and required skillsets.
  • Add Project: Click "+ Add Project" to create a new project. Enter project ID, name, effort estimates (BA, Planner, Dev in person-months), priority, parent program, and required skillsets.
  • Edit Project: Click on any cell in the table to edit it directly. Changes are highlighted in yellow.
  • Delete Project: Click the trash icon next to a project to remove it.
  • Change Priority: Use the up/down arrows to adjust project priority (lower number = higher priority).
  • Save Changes: Click "Save Changes" to persist your edits. Click "Discard Changes" to revert.
  • Filter by Program: Use the program filter dropdown to view projects from a specific program.

Programs Subtab

  • View Programs: See all programs (project groupings) with their assigned colors.
  • Add Program: Click "+ Add Program" to create a new program with a name and color.
  • Edit Program: Click on the name to edit it, or click the color picker to change the program color.
  • Delete Program: Click the trash icon to remove a program (note: projects assigned to this program will still exist).

Timeline Subtab

  • View Results: After running the model, see when each project is scheduled to run, its duration, and assigned team members.
  • Sort Options: Click "Sort by Program" or "Sort by Start Date" to reorder the timeline view.
  • Filter by Program: Use the program filter to view timeline for specific programs.

Unallocated Subtab

  • View Failed Projects: See projects that couldn't be scheduled due to resource constraints or other issues.
  • Understand Why: Read the explanations for why each project failed to schedule and identify bottlenecks.

Portfolio Settings Tab

Configure planning parameters that control how the scheduling algorithm works:

  • Planning Start/End: Set the time window for planning (leave end date blank for open-ended planning).
  • KTLO Percentages: Set "Keep The Lights On" allocation percentages by role (time reserved for ongoing work).
  • Max Concurrent Projects: Limit how many projects a person can work on simultaneously by role.
  • Planner Cap: Maximum percentage of a planner's time that can be allocated to a single project per month.
  • Effort Curves: Define how effort is distributed over project duration (uniform, bell curve, front-loaded, etc.).
  • Priority-Based Scheduling: Enable to schedule higher priority projects first.
  • Overbooking Tolerance: Allow slight overbooking (e.g., 20% means 120% allocation is acceptable).

People Tab

Staff Subtab

  • View Team: See all people organized by their roles (BA, Planner, Dev).
  • Add Person: Click "+ Add Person" to add a new team member. Specify name, roles, active status, start/end dates, skillsets, and preferred programs.
  • Edit Person: Click the edit icon on any person card to modify their details.
  • Delete Person: Click the trash icon to remove a person (be careful - this affects resource availability).
  • Skillsets: Add skills to match project requirements (e.g., "front-end", "back-end", "security").
  • Preferred Programs: Optionally specify which programs a person prefers to work on.

Resource Allocation Subtab

  • View Heatmap: After running the model, see a color-coded heatmap of resource allocation by person and month.
  • Expand/Collapse: Click on role names to expand and see individual people, or click person names to see their project-level allocations.
  • Color Coding: Blue shades indicate allocation levels (light = low, dark = high). Red indicates overallocation (>100%).
  • KTLO Tracking: See KTLO allocations separately from project work.

Modeller Tab

Run the capacity planning algorithm:

  1. Select a portfolio from the dropdown
  2. Ensure your projects, people, and settings are configured
  3. Click "Run Model" to start the scheduling algorithm
  4. Monitor job status in the "Recent Jobs" table
  5. When complete, view results in the Projects ‚Üí Timeline and People ‚Üí Resource Allocation tabs

Files Tab

Direct access to portfolio input and output files:

  • Input Files: projects.csv, people.json, config.json, programs.csv
  • Output Files: project_timeline.csv, resource_capacity.csv, unallocated_projects.md
  • Click on any file to download or view it

Best Practices

  • Start with Sample Data: Create a new portfolio to get familiar with the data structure before customizing.
  • Set Realistic KTLO: Reserve 10-20% of time for ongoing operational work.
  • Use Priorities: Assign priority numbers to projects so critical work gets scheduled first.
  • Match Skills: Ensure people have skillsets that match project requirements.
  • Review Unallocated: After running the model, check the Unallocated tab to understand capacity constraints.
  • Iterate: Adjust priorities, effort estimates, or team composition based on results, then re-run the model.

Tips

  • Changes to projects, people, and settings are saved immediately - no need to run the model to persist edits.
  • The model must be re-run after making changes to see updated timeline and allocation results.
  • Use program colors to visually organize related projects.
  • The resource allocation heatmap helps identify overallocated team members.
  • Project IDs must be unique within a portfolio.

Troubleshooting

  • Projects not scheduling: Check if you have enough people with the required skills and availability.
  • Overallocation: Increase team size, reduce project scope, extend timelines, or adjust KTLO percentages.
  • Model errors: Verify all required fields are filled in projects.csv, people.json, and config.json.
  • Missing results: Ensure the model completed successfully by checking the Recent Jobs table.

Data Format Notes

  • Dates: Use ISO format (YYYY-MM-DD) for all dates.
  • Effort: Specified in person-months (e.g., 2.5 = 2.5 months of one person's time).
  • Skillsets: Separate multiple skills with semicolons or use JSON array format.
  • Programs: Optional - projects can exist without being assigned to a program.
